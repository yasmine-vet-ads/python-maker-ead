Módulo 1: Primeiros Passos e o "Modo Pensar" do Python

Seja muito bem-vindo! Programar não é decorar fórmulas matemáticas difíceis, é apenas aprender a conversar com a máquina. Pense no código como uma receita de bolo detalhada: o computador apenas lê linha por linha e executa exatamente o que você mandou.

Por que o mercado escolheu o Python?

  • Sintaxe Limpa: O código parece inglês fluente. É feito para humanos lerem sem esforço, reduzindo erros.
  • Superpoderes Tecnológicos: É a linguagem número 1 para Inteligência Artificial (IA), Automação de Sistemas e Internet das Coisas (IoT).

Conceitos na Prática: Variáveis e Saída de Dados

Variáveis funcionam como "caixinhas" com etiquetas na memória para guardar dados temporários. O comando básico para mostrar qualquer informação guardada na tela do computador é a função print(). Veja esse exemplo simulando o prontuário de uma clínica veterinária:

nome_paciente = "Bruce Brian"  # Criamos uma variável de Texto (String)
idade = 5                      # Criamos uma variável Numérica Inteira (int)

print("Paciente:", nome_paciente)
print("Idade registrada:", idade, "anos")
  • nome_paciente = "Bruce Brian": O sinal de igual = serve para "guardar" a informação dentro da caixinha. Usamos aspas porque é um texto livre (String).
  • idade = 5: Números puros não levam aspas. Assim, o Python entende matematicamente que é um Inteiro (int) e permite fazer contas com ele futuramente.
  • print(...): É a função de exibição. A vírgula dentro dos parênteses serve para unir textos fixos com os dados guardados nas variáveis.

Módulo 2: Tomando Decisões e Repetições

Para darmos inteligência ao código, precisamos que ele tome decisões sozinho usando uma lógica condicional de "Se isso acontecer, faça a tarefa A; Senão, faça a tarefa B".

1. Estruturas Condicionais (if / else)

temperatura = 39.5

if temperatura > 39.2:
    print("Alerta clínico: O animal está com febre!")
else:
    print("Temperatura dentro da normalidade.")
  • if temperatura > 39.2: Significa "SE a temperatura for maior que 39.2". Os dois pontos (:) abrem o bloco de ação.
  • Recuo (Indentação): Note que as funções print estão empurradas 4 espaços para a direita. Isso é obrigatório no Python! É assim que ele sabe quais linhas dependem da decisão do if.
  • else: Significa "SENÃO". Se a condição do if for falsa, o Python ignora ela e executa o bloco do else.

2. Listas e Loops de Repetição (for)

E se precisarmos analisar dezenas de dados em sequência? Criamos uma Lista usando colchetes [] e aplicamos um Loop for para repetir a leitura linha por linha.

sintomas = ["tosse", "apatia"] # Criamos a lista com dois sintomas
sintomas.append("febre")       # Adicionamos dinamicamente um terceiro item

for sintoma in sintomas:
    print("Foi detectado o sintoma:", sintoma)
  • .append("febre"): Pega a lista existente e injeta um novo elemento bem no final dela.
  • for sintoma in sintomas:: É o laço. O Python roda o print três vezes seguidas, trocando a palavra automaticamente a cada volta.

Módulo 3: Organizando a Casa com Funções e Módulos

Quando o nosso sistema começa a crescer, escrever códigos soltos gera bagunça. Para organizar, empacotamos blocos de regras dentro de Funções reutilizáveis.

Funções Customizadas (def) e Bibliotecas (import)

import time  # Importamos a biblioteca nativa de tempo do sistema

def calcular_dosagem(peso_animal):
    dosagem = peso_animal * 0.5  # Regra de negócio: 0.5mg por quilo
    return dosagem

print("Acessando banco de dados...")
time.sleep(2)  # Faz o programa pausar por 2 segundos reais

resultado = calcular_dosagem(10) # Chamamos a função passando o peso 10
print("Dose calculada com segurança:", resultado, "mg")
  • import time: Abre a caixa de ferramentas do Python e traz funções prontas de relógio para o seu código.
  • def calcular_dosagem(peso_animal):: O comando def cria a função. O dado dentro dos parênteses é o parâmetro recebido de fora.
  • return dosagem: É a devolução. Entrega o valor calculado para a variável de saída do script principal.
  • time.sleep(2): Pausa a execução do terminal, excelente para simular carregamentos ou cronometrar sensores.

Módulo 4: Projeto Integrador Maker (Semáforo de Pedestres)

Agora vamos unir tudo! A Cultura Maker prega o "faça você mesmo", transformando linhas de código em ações no mundo físico através de microcontroladores (como o ESP32) programados em MicroPython.

Lógica de Hardware Aplicada

  • Setup de Pinos: Configurar pinos como Saída (Pin.OUT) serve para injetar energia (acender LEDs). Configurar como Entrada (Pin.IN) serve para monitorar sensores ou botões.
  • O Papel Crítico dos Resistores: LEDs queimam se receberem energia total direta da placa. O resistor funciona como um "pedágio", diminuindo a corrente elétrica para proteger a lâmpada. No botão, ele estabiliza o sinal contra ruídos estáticos da atmosfera.

🔌 Passo a Passo para o Aluno Simular no Wokwi

  1. Abra o site wokwi.com, role a página inicial e escolha a placa ESP32 rodando MicroPython.
  2. Clique no botão azul de "+" (Add a new part) para puxar os componentes: LEDs (Vermelho, Amarelo, Verde), resistores de 220 ohms e 1 botão do tipo Pushbutton.
  3. Conecte a perna positiva do LED no pino correspondente do código (ex: Pino 32) e a perna negativa no pino GND (Terra) passando pelo resistor protetor.
  4. Substitua o código da aba main.py pelo script abaixo e aperte o botão verde de **Play** do simulador!
